# Git Setup Commands for Snowflake Semantic Model Pipeline

# Step 1: Initialize Git repository (if not already done)
git init

# Step 2: Add remote repository
git remote add origin https://github.com/manideepv28/anon_pipeline_dataset.git

# Step 3: Add all files to staging
git add .

# Step 4: Create initial commit
git commit -m "Initial commit: Complete Snowflake Semantic Model Pipeline

- Added one-command setup script (setup_pipeline.py)
- Added comprehensive configuration (config.py)
- Added complete documentation (README.md, DEPLOYMENT_GUIDE.md)
- Added all pipeline scripts in scripts/ directory
- Added semantic model YAML definition
- Added requirements.txt for dependencies
- Added .gitignore for proper file management
- Added MIT license

Features:
- One-command deployment: python setup_pipeline.py
- Automatic data loading from split CSV files
- Data integrity verification
- Semantic model creation from YAML
- Complete Snowsight setup instructions
- Comprehensive error handling and logging"

# Step 5: Push to GitHub
git branch -M main
git push -u origin main

# Alternative: If you want to force push (use carefully)
# git push -u origin main --force

# Step 6: Verify the push
git status
git log --oneline -5
